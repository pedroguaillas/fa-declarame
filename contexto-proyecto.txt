## BASE DE DATOS

### Tablas

```
roles
  id, name, slug, description, timestamps

plans
  id, name, slug, description, price (decimal 10,2),
  max_employees (int), is_active (bool), timestamps

permissions
  id, name, slug, description, timestamps

model_entities
  id, name, slug, description, timestamps

model_permissions
  id, role_id (FK), permission_id (FK), model_entity_id (FK), timestamps
  UNIQUE(role_id, permission_id, model_entity_id)

users
  id, role_id (FK), admin_id (FK users nullable),
  name, email, email_verified_at, password,
  is_active (bool), remember_token, timestamps

subscriptions
  id, user_id (FK), plan_id (FK), created_by (FK users),
  start_date (date), end_date (date), is_active (bool), notes, timestamps
```

### Datos seed por defecto

**Roles:** super_admin, admin, employee

**Planes:** Básico ($29.99, max 5 emp), Pro ($79.99, max 20 emp)

**Permisos:** view, create, edit, delete, assign

**Módulos:** permissions, models, roles, users, plans, subscriptions

**Permisos por rol:**
- super_admin → TODO en TODOS los módulos
- admin → view/create/edit/delete en users y subscriptions
- employee → solo view en users

**Usuarios seed:**
- superadmin@admin.com / password → super_admin
- admin@admin.com / password → admin
- empleado@admin.com / password → employee (admin_id = admin)

## MODELOS ELOQUENT

### User.php
```php
// Relaciones
role()                       → BelongsTo Role
admin()                      → BelongsTo User (admin_id)
employees()                  → HasMany User (admin_id)
subscriptions()              → HasMany Subscription
createdSubscriptions()       → HasMany Subscription (created_by)
activeSubscriptionRelation() → HasOne Subscription (is_active=true, end_date>=now) ← eager load

// Helpers
isSuperAdmin() / isAdmin() / isEmployee() → bool
activeSubscription()         → ?Subscription (método, no relación Eloquent)
hasActiveSubscription()      → bool
hasPermission(permSlug, modelSlug) → bool (super_admin siempre true)
```

### Subscription.php
```php
user()      → BelongsTo User
plan()      → BelongsTo Plan
createdBy() → BelongsTo User (foreign: created_by)
isExpired() → bool
isValid()   → bool (is_active && !isExpired)
```

## MIDDLEWARES

### CheckSubscription (alias: check.subscription)
- super_admin → pasa siempre
- admin → necesita suscripción activa (is_active=true Y end_date >= hoy)
- employee → necesita que su admin tenga suscripción activa
- Si falla → logout + redirect login con error

### RoleMiddleware (alias: role)
- Uso: Route::middleware('role:super_admin')
- Acepta múltiples: 'role:super_admin,admin'
- Falla → abort(403)

### HandleInertiaRequests
Comparte globalmente:
```js
{
  auth: { user: { id, name, email, role, has_active_subscription } },
  flash: { success: string|null, error: string|null },
  ziggy: Config & { location: string }
}
```

## LÓGICA DE NEGOCIO

### Suscripciones
1. Super Admin asigna plan + fechas a un Admin
2. Al crear nueva suscripción → las anteriores del mismo usuario quedan is_active=false
3. Suscripción válida: is_active=true AND end_date >= hoy
4. Admin con suscripción vencida → él y sus empleados no pueden entrar
5. Se mantiene historial completo (nunca se eliminan, solo se desactivan)
6. toggleActive: al activar una, desactiva las demás del mismo usuario

### Empleados
- Tienen admin_id apuntando al Admin que los creó
- Al crear empleado se valida max_employees del plan activo del Admin
- Si Admin no tiene suscripción activa → abort(422)

### Permisos
- Combinación única: rol + permiso + módulo (tabla model_permissions)
- super_admin bypasea siempre: hasPermission() retorna true
- Uso: $user->hasPermission('view', 'users')

### UX Pattern — Tablas con Dialogs (NO formularios en página)
Todas las páginas siguen el mismo patrón:
1. Header con título + icono + descripción + botón "Nuevo X" (justify-between)
2. Tabla con datos en rounded-lg border border-border bg-card
3. Dialog para CREAR (abre con openCreate(), resetea form)
4. Dialog para EDITAR (abre con openEdit(item), carga datos)
5. Dialog para CONFIRMAR ELIMINAR
EXCEPCIÓN: Roles/Form.vue tiene página propia por la matriz de permisos.

### Switch — siempre usar :model-value
```vue
<Switch
    :model-value="form.is_active"
    @update:model-value="(val) => form.is_active = val"
/>
```

### Select — siempre con w-full en el trigger
```vue
<SelectTrigger class="w-full">
```

### Links de navegación — siempre Link de Inertia (NUNCA <a href>)
```vue
import { Link } from '@inertiajs/vue3'
<Link :href="route('dashboard')">Dashboard</Link>
```

### Navegación programática
```vue
import { router } from '@inertiajs/vue3'
router.visit(route('users.index'))
router.patch(route('users.toggle-active', id))
router.delete(route('users.destroy', id))
```

### Formularios Inertia
```ts
const form = useForm({ campo: '' })
form.post(route('ruta.store'))
form.put(route('ruta.update', id))
form.patch(route('ruta.update-info'))
// onSuccess siempre cierra dialog y resetea form
```

### Flash messages
Backend: return back()->with('success', 'mensaje') o ->with('error', 'mensaje')
Frontend: FlashMessage.vue captura automáticamente, auto-desaparece en 4s
Está montado en AppLayout.vue fuera del slot.

### DatePicker
Usa @internationalized/date, NO date-fns para el calendario.
Emite string 'yyyy-MM-dd' via @update:modelValue.
```vue
<DatePicker v-model="form.start_date" placeholder="Seleccionar fecha" />
```

### Formateo de fechas en tablas
Usar date-fns con locale español:
```ts
import { format, parseISO } from 'date-fns'
import { es } from 'date-fns/locale'
format(parseISO(date), 'dd MMM yyyy', { locale: es })
```

### Formateo de moneda
```ts
new Intl.NumberFormat('es-EC', { style: 'currency', currency: 'USD' }).format(amount)
```

### TypeScript — ziggy.d.ts

## NOTAS CRÍTICAS PARA LA IA

1. **Docker first**: NUNCA sugerir instalar PHP, Composer o Node en el host WSL.
   Todo va con prefijo `sail`.

2. **Tailwind v4**: NO tailwind.config.js. Configuración en app.css con
   `@import "tailwindcss"` y `@theme {}`. NO @layer base/components/utilities.

3. **TypeScript estricto**: Todos los componentes usan `<script setup lang="ts">`.
   Siempre tipar props con defineProps<{}>().

4. **Shadcn-vue**: Componentes en @/components/ui/. NUNCA colores hardcodeados.
   SIEMPRE variables de tema. Switch usa :model-value. Select usa w-full en trigger.

5. **Inertia**: NO hay API REST. Respuestas son Inertia::render() o redirect()->with().
   Forms usan useForm() de @inertiajs/vue3.

6. **Links internos**: SIEMPRE <Link> de Inertia. NUNCA <a href> para rutas internas
   (causa recarga completa de página y rompe la experiencia SPA).

7. **Pattern UX**: SIEMPRE tabla + dialogs para CRUD. NUNCA formulario inline en la
   página. EXCEPCIÓN: Roles/Form.vue tiene página propia por la matriz de permisos.

8. **Laravel 13**: Route model binding, Form Requests para validaciones complejas,
   Policies para autorización, Resources para respuestas consistentes.

9. **Flash messages**: SIEMPRE return back()->with('success'/'error', 'mensaje').
   FlashMessage.vue los captura automáticamente desde props.flash.

10. **Slugs de sistema protegidos**:
    Roles: super_admin, admin, employee
    Módulos: permissions, models, roles, users, plans, subscriptions
    Deshabilitar botón eliminar en UI cuando systemSlugs.includes(item.slug).

11. **Paginación**: ->paginate(15) en controllers. Paginación inline en Index.vue
    con goToPage(url) usando router.visit(url, { preserveScroll: true }).

12. **DashboardController**: Es invokable (__invoke), usa match() por rol.
    Cada rol retorna Inertia::render() con su vista y props específicas.

13. **activeSubscriptionRelation vs activeSubscription**:
    - activeSubscriptionRelation() → HasOne, usar para with(['activeSubscriptionRelation.plan'])
    - activeSubscription() → método que retorna ?Subscription, usar en lógica PHP

14. **Tema**: Script inline en app.blade.php es CRÍTICO para evitar flash.
    useTheme() tiene listener MediaQuery para modo sistema en tiempo real.

15. **route() global**: NO importar en cada componente. Está en ziggy.d.ts global.
    Si falla: Ctrl+Shift+P → TypeScript: Restart TS Server en VSCode.

16. **Caché Vite**: Si aparece ERR_ABORTED 504 (Outdated Optimize Dep):
    rm -rf node_modules/.vite && sail npm run dev
